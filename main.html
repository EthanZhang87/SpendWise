Introduction

Project Overview

SpendWise is a project for my CS2340 class. It is a mobile app that focuses on simplifying the travel itineraries by allowing easy creation and management of solo and group travel itineraries. This app is perfect for those who are tired of unorganized trips and messy excel sheets! SpendWise simplifies the process, and allows users to live in the moment. The app uses Java, as well as Android Studio. For the database, we implemented Firebase in order to take advantage of Firestore and its realtime database, as well as its authentication features to allow for login/logout features. Furthermore, the app utilizes Ollama in order to power its AI ChatBot. The entire project took most of the semester to build, and was built using the agile methodology, as we split the project into 5 distinct sprints.

Design & Architecture

Architecture

For our architecture, we utilized the MVVM architecture to ensure clean separation in our code. MVVM helped us keep our data, UI, and business logic independent from each other. The Model is responsible for the data and main application logic, the View is responsible for everything the user sees, and the ViewModel sits in between, kind of acting as a middle-man between the View and Model. This separation made our codebase easier to test, reduced duplication, and allowed us to update UI behavior without touching the underlying logic. We also used the Strategy pattern for weekly and monthly budgeting periods and a Factory pattern for producing charts.

User Interface

Visual Tour

Below is a guided tour through the key screens of SpendWise.

Dashboard

The central hub displays an overview of expenses and budgets at a glance. It uses graphs and charts to visualize this data.

Expense Log

Track all expenses with category tags, timestamps, and participant information.

Budget Management

Set and monitor budgets along with group saving goals.

Savings Circle

A screen that showcases all the different group saving circles currently active.

AI Chatbot Assistant

An AI-powered assistant that provides the user with relevant and useful information regarding their budget, expenses, etc.

Design Consistency

We maintained visual consistency throughout the app by establishing a unified design system. All screens share the same colors, fonts, and overall styling.

Theming

We have both light and dark modes to accommodate different user preferences.

Navigation

User navigation utilizes a bottom navigation bar for primary features and a top app bar for some other actions like logging out or changing the date. The navigation structure is very minimal to minimize actions needed to reach any feature.

Components

We used reusable components (cards, buttons, input fields, charts) that maintain consistent behavior and appearance across screens.
